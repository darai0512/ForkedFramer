# 修正済

- キャビネット内の「ローカルストレージ（File System Access API / 保存フォルダ指定）」セクションおよび補助データ保存場所ラジオボタンを完全削除（関連スクリプト変数・関数・importも整理）
- `Drawer.svelte` を修正し、すべてのスワイプモーダルの一番下に「閉じる」ボタンを追加（モーダル外クリックによる閉じる動作は引き続き有効）
- リロード時の初期フォーカスを修正: ページ遷移時と同じ `focusToPage(0, 1, true)` を使用することで、1ページ目がキャンバスに正しくセンタリングされるよう改善（従来のハードコード `translate: [150, 0]` を廃止）
- 広告用コンポーネント（AdContainer, AdPanel）をメイン画面右側から削除
- 描画領域を広げるため、グローバルヘッダーと付帯機能（プロフィール、認証等）を完全削除
- 「元に戻す・やり直す」ボタンと「吹き出し表示」のプルダウンのみを抽出し、キャンバスの拡大縮小等に影響されないよう画面左上に固定配置
- ダウンロードボタンの配置を右側から左側のメニュー群へ移動
- 左上に新しく固定したUIと被らないよう、既存の左側ボタン（定規・About・ダウンロード）の縦の並び位置を1段階ずつ下に調整
- 左側に並んでいたメインメニューの各種ボタンサイズを従来の半分（縦横60px・固定サイズ）に縮小
- 画面左右中央に配置したページ移動ナビゲーションボタンとの重なりを防ぐため、上記メニューボタン群をすべて「画面左下」基準へ移動し、2列のすっきりした二段組レイアウト（Col:0〜1、Row:0〜2）に再配置
- 最初のページでは「前ページ」へのボタンを自動で非表示化し、最終ページから「次ページ」へ移動しようとした場合は自動で新規ページを挿入して遷移する仕様を追加
- キャンバスのページ閲覧部直下に表示されていた「新しいページを挿入」アイコンを非表示化（※コピー＆ペースト処理は維持）
- コントロールパネル（版形）にあった「拡大率」調整ブロックをパネル内から削除し、画面左上のグローバルナビへ直接組み込み（※設定範囲も50%〜200%へ調整）
- 画面左上のグローバルナビについて、「アンドゥ・リドゥ・吹き出し設定」を1段目、「キャンバス拡大率」を2段目に配置するようレイアウトを改修し開閉式に
- 画面左下にあった「新規ドキュメント（本のマーク）」ボタンを削除
- キャビネット内の「ページ作成（フォルダ内でのファイル新規作成）」ボタンの挙動を変更し、作成後自動的にそのファイルをキャンバス上に開いてすぐに編集を始められる仕様に改善
- 左下ボタンから展開されるドロワー（スワイプイン・モーダル）のZインデックスを調整し、開いた際に上部ツールバーや左右のページめくりボタンに隠れず正しく最前面に表示されるように修正
- ダウンロードメニュー（左下のダウンロードボタン経由）に含まれていた、ログインやオンライン連携が必要な機能（「アップスケールしてダウンロード」「まんがファームへの公開」など）を削除し、ローカルでのエクスポート操作のみに整理
- ページアイコン（マーク、コピー、吹き出し等）をサイズ変更し、キャンバス画面下部からページの右側外側余白へ列として配置
- キャンバス上の「吹き出し生成」アイコンボタンをページ右側のページアイコン列の一番上に移動し、常に右上に配置。
- 画面右側に並ぶページアイコン群のうち、「ゴミ箱」アイコンのみ誤操作防止のため列全体とは切り離し、ページ右下端へ個別に配置変更
- クリップボードからの画像ペースト（Ctrl+Vなど）時の挙動を改善。マウスカーソルがキャンバス外にある場合でも画面中央にオブジェクトとしてペーストできるようにし、Chromium系ブラウザでコピーした画像データの読み込みに対応
- コレクション内の画像ビュワーを一律の大きさへの正方形(aspect-ratio: 1/1)サムネイルによる5列グリッド配置へと改修
- コレクション間での画像・メディアのDrag&Dropによる移動（所属の変更）に対応
- 右下のゴミ箱ボタンからページを削除した直後、一番最初のページ（インデックス0）へ自動的にフォーカスが戻るように改善
- グローバルナビゲーションの2列の幅を揃えるため、アンドゥ・リドゥボタンの横幅を調整（50px → 40pxへ縮小）
- 素材集機能からオンライン関連を削除
  - 素材集機能: 画像登録後、
  - 別のコマ・ページにドラッグ＆ドロップ 素材集パネル上に並んでいる使いたい画像を、そのままキャンバス上の 入れたいコマに向かってドラッグ＆ドロップ してください。これでローカルから読み込む手間なく直接コマに適用されます。
  - ふきだし/オブジェクトとしてポンと置く 素材集パネル上の画像を（ドラッグではなく）ただクリックするだけで、今見ているページの空いている場所に「画像オブジェクト（吹き出し扱い）」として即座に配置されます。背景がないキャラクター画像などをポンポン置きたいときに便利です。 
- コマや吹き出しにドラッグ＆ドロップされたメディア画像を自動的に「素材集」にも追加・保存
- コマクリック時の「＋」ボタンで表示される画像挿入メニューへ「素材集」を移す
  - 他のコレクションを開いた際に前を閉じるAutoCollapse機能、末尾の「＋ 新しいコレクション」追加用ボタンの実装
- 左下「現在セーブ禁止」トグルを削除し、「キャビネット」内へ移動
- キャビネットモーダル内にすべてのローカルストレージデータを空にする初期化ボタンを追加
- キャビネットモーダル内から「クラウドストレージ」関連の文言を削除
- ログイン関連機能の全削除
- コレクション間の画像D&D移動時に画像が消失するバグを修正（Entry型のNodeIdを誤って`.id`参照していた問題）
- 素材集のアコーディオンをAutoCollapseからマニュアル開閉に変更（複数コレクション同時展開可能に）
- コレクション並び替えD&Dのファイルシステム永続化ロジックを除去（エントリ破壊の危険があったため。並び順はセッション内のみ保持）


- File System Access API (showOpenFilePicker) がブラウザに存在するかどうかです。
  - ✅ Chrome/Edge（デスクトップ）→ 利用可能
  - ❌ Firefox / Safari / HTTPSではないサイト → 利用不可
  - ❌ HTTP環境（セキュアコンテキスト外）→ 利用不可
